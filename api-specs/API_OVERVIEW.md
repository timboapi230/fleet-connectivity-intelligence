# Aeris IoT Accelerator — API Overview

> **For Contest Judges:** These 7 OpenAPI specs (.yaml) are the real API contracts from the Aeris IoT Accelerator platform — the same platform Telenor IoT uses in production to manage SIM connectivity for Geotab devices. All data shown in the Add-In demo is **realistic mock data** generated from these specifications. No live customer data is exposed.

---

## The Gap These APIs Fill

Geotab knows **where** a vehicle is and **that** it has weak signal. These Aeris APIs know **why** — which tower, which carrier, which errors, and how much data is left. Fleet Connectivity Intelligence is the first tool to combine both.

---

## The 7 APIs

| # | API | Method | What It Returns | Used In |
|---|-----|--------|----------------|---------|
| 1 | **Subscription Location** | `GET` | Cell tower identity (MCC/MNC/LAC/Cell ID), radio type (LTE/WCDMA/GSM) | Coverage Map, Fleet Map, 3D Command Center |
| 2 | **Consumer Connectivity** | `GET` | Per-SIM data usage (upload/download), plan limit, remaining balance | Data Usage |
| 3 | **Device Reconnect** | `POST` | PDP context success/failure, attach/detach events, reconnection history | Problem Alerts, Diagnostics |
| 4 | **Incident Management** | `GET` | Open tickets, severity, affected SIM count, resolution status | Problem Alerts |
| 5 | **Organization Signaling** | `GET` | Fleet-wide KPIs: session success rate, location update rate, error totals | Network Diagnostics |
| 6 | **Subscription Signaling Usage** | `GET` | Per-SIM signaling event counts (attaches, detaches, location updates) | Diagnostics breakdowns |
| 7 | **Subscription Signaling Events** | `GET` | Detailed error logs: AUTH_FAILURE, PDP_REJECT, ATTACH_REJECT, GPRS_DETACH | Errors & Export |

**Authentication:** All APIs use OAuth2 (Bearer token) over HTTPS.  
**Base URL:** `https://iot-api.aeris.com/iot/api/`  
**Identifiers:** SIMs are queried by IMSI, MSISDN, or ICC.

---

## Current Model vs. Roadmap

**Today (Pull — GET):** The Add-In retrieves data on demand. Geotab can make batch API requests to view connectivity data filtered by country, carrier, or vehicle.

**Future (Push — Webhooks & Write-Back):**
- **Aeris → Geotab:** Webhook events (error threshold exceeded, data cap approaching, incident opened) could auto-create Geotab Exception Events — no polling required.
- **Geotab → Aeris:** Geotab rules could trigger carrier profile switches when a vehicle enters a known dead zone.
- **Batch Analytics:** Scheduled pulls into Geotab Data Connector for Power BI / Excel / BigQuery dashboards.
